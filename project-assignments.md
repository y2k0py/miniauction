# Software Engineering — Final Project Assignments

## Initial Assignment (Exercise 6)

We are starting work on final projects:

- A software system project may be completed in groups (maximum 3 people).
- The evaluation will cover the complete project along with a working program/system and full project documentation (project description, operating instructions, UML diagrams, screenshots, and other materials that will be discussed in subsequent lectures).

### Tasks for today

1. Dividing students into groups
2. Group work on the project topic and its functionalities
3. The project must include at least 5 distinct functionalities
4. Project implementation — any programming language

**Confirmation of completed work:** Please submit the composition of project groups and the project topic.

## Exercise 13

### Tasks for today's class

In the last lecture we discussed the complexity of applying security measures. We talked about four principles:

1. **Principle of natural user interaction**
2. **Principle of horizontal and vertical consistency**
3. **Principle of least privilege**
4. **Principle of default deny access**

Please describe specific solutions related to the above principles in the context of your project.

### Preparation for upcoming exercises

Starting from the next exercises, we begin presenting completed projects. You must prepare:

- **Full technical documentation** (all points developed from the beginning of the project)
- **Working application, program, system, etc.**
- **Multimedia presentation** supporting a live demonstration of the project

During the exercises, each group/individual will have approximately 10 minutes to present the working project. The evaluation will cover the points mentioned earlier, the complexity of the presented project, and the level of involvement of individual group members in completing the project.
